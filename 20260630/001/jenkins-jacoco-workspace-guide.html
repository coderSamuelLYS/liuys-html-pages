登录机器后先准备目录

以下命令在 Jenkins 执行机器上执行，不要在本机执行。

jenkins-node ~/workspace 
$ mkdir -p /data/jenkins/workspace/autotest-api-feat-service-address-profile
$ cd /data/jenkins/workspace/autotest-api-feat-service-address-profile
$ pwd
$ ls -la
!
不要直接用默认目录 不要改 /data/jenkins/workspace/autotest-api。默认目录可能被其他构建使用，独立目录更容易确认代码版本。

执行后检查

  • ✓pwd 正确输出应是 /data/jenkins/workspace/autotest-api-feat-service-address-profile。
  • ✓目录可写ls -la 能正常输出，后续能 clone 文件。

第一次 clone 分支代码

目录为空时使用这一组命令。执行前先确认本地分支已经 push 到远端。

jenkins-node workspace/autotest-api-feat-service-address-profile 
$ cd /data/jenkins/workspace/autotest-api-feat-service-address-profile

$ git clone git@git.firstshare.cn:autotest/fs_autotest_api.git .
$ git checkout feat/service-address-profile
$ git pull --ff-only origin feat/service-address-profile

$ git remote -v
$ git branch --show-current
$ git log -1 --oneline
$ git status --short

分支有新提交时更新目录

目录已经 clone 过时，不要重新 clone。用这组命令更新到远端最新分支。

jenkins-node workspace/autotest-api-feat-service-address-profile 
$ cd /data/jenkins/workspace/autotest-api-feat-service-address-profile

$ git status --short --branch
$ git fetch origin
$ git checkout feat/service-address-profile
$ git pull --ff-only origin feat/service-address-profile

$ git branch --show-current
$ git log -1 --oneline
$ git status --short

看到本地有改动时

如果 git status --short 有输出，先不要继续 pull。确认是不是上次构建生成的报告或临时文件。

jenkins-node 只查看，不删除 
$ cd /data/jenkins/workspace/autotest-api-feat-service-address-profile
$ git status --short
$ git diff --stat

常见异常处理

只列机器操作相关问题。

git clone 提示没有权限
说明 Jenkins 机器上的 SSH key 没有访问 git.firstshare.cn:autotest/fs_autotest_api.git 的权限。先在机器上执行 ssh -T git@git.firstshare.cn。如果不能认证，需要给机器配置可访问该仓库的 SSH key。
git pull 提示不能 fast-forward
先执行 git status --short 和 git log --oneline --decorate -5 查看目录状态。不要直接覆盖默认目录；这个独立 workspace 可以确认没有需要保留的改动后再重新建目录。
Jenkins 构建仍然像旧代码
登录机器执行 cd /data/jenkins/workspace/autotest-api-feat-service-address-profile && git log -1 --oneline。同时看 Jenkins Console Output 是否有 SCM checkout 把目录切回 master。
本机能打开 Jenkins，但本机不能跑 jacoco
这是正常现象。本机解析不到 jacoco 内部 LB。验证 jacoco 自动化应该放到 Jenkins 机器上跑。
